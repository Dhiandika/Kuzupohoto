# 🛠️ Photobooth Web App: Agent Instructions & Detailed Workflows

## .skill (Required Agent Capabilities)
Agent harus memiliki pemahaman mendalam pada area teknis berikut untuk mengeksekusi proyek ini dengan sukses:

1.  **Astro & Vanilla JS State Architecture:**
    *   Mampu membuat *Single Page Application* (SPA) *feel* di dalam satu file `.astro` (contoh: `booth.astro`).
    *   Menguasai DOM Manipulation (menyembunyikan/menampilkan `<section id="step-1">`, dll) tanpa *framework reactive* (React/Vue), karena Astro secara *default* mengirimkan HTML statis.
    *   Mampu menyimpan dan mengelola *state global* (seperti konfigurasi *session*, array foto sementara) menggunakan `window` object, Closures, atau `sessionStorage`.

2.  **Advanced Tailwind CSS (Neo-Brutalism Styling):**
    *   Mampu memodifikasi `tailwind.config.mjs` untuk menciptakan tema *Neo-Brutalism* (Referensi 1).
    *   Keahlian membuat:
        *   `box-shadow` solid dan asimetris (contoh: `4px 4px 0px 0px rgba(0,0,0,1)`).
        *   Border hitam tebal (`border-4 border-black`).
        *   Efek *hover* yang menghilangkan shadow dan menggeser elemen (memberi kesan tombol fisik ditekan).
    *   Mengimplementasikan font bergaya *chunky* atau retro (seperti *Lilita One* atau *Press Start 2P*) secara konsisten.

3.  **MediaDevices API & Camera Handling:**
    *   Mampu meminta izin kamera (`navigator.mediaDevices.getUserMedia`) dengan parameter resolusi yang dioptimalkan (`{ video: { width: 1920, height: 1080, facingMode: "user" } }`).
    *   Keahlian menangani masalah "Video Gepeng" (menggunakan CSS `object-fit: cover`) dan membalik video secara horizontal (`transform: scaleX(-1)`) agar kamera terasa seperti cermin (*mirroring*).
    *   Mampu merilis (stop) track kamera saat tidak dibutuhkan untuk menghemat baterai perangkat.

4.  **HTML5 Canvas Mastery (The Core Engine):**
    *   Mampu mengekstrak *frame* tunggal dari tag `<video>` yang sedang berjalan ke dalam `<canvas>`.
    *   Menguasai **Algoritma Image Cropping/Scaling di Canvas**: Menggunakan ke-9 parameter `ctx.drawImage(image, sx, sy, sWidth, sHeight, dx, dy, dWidth, dHeight)` untuk memotong foto (16:9) agar pas ke dalam kotak *frame layout* (misal 3:4 atau 1:1) tanpa merusak *aspect ratio*.
    *   Mampu menerapkan CSS/Canvas Filters (`ctx.filter`) seperti *Grayscale*, *Sepia*, dll sebelum menggambar gambar ke *canvas final*.
    *   Menggambar teks (Caption) dengan custom font di atas *canvas final* (`ctx.fillText`).

## .Rules / Workflows (Detailed Application Logic & Flow)

Proyek ini menggunakan alur **4 Langkah (Stepper)** yang terinspirasi dari "Jelly's Photobooth" (Referensi 2), namun dibalut dengan UI ceria ala "Pixenze" (Referensi 1). Seluruh proses berjalan di sisi *Client* (tanpa Backend).

### Fase 0: Keamanan & Akses (The Gatekeeper)
*   **Halaman:** `index.astro`
*   **Fitur:** UI Input Passcode bergaya Arcade.
*   **Logika:** 
    *   Jika `input_value === "TUMBAL_2025"` (atau PIN yang ditentukan), simpan `sessionStorage.setItem('auth', 'true')` lalu `window.location.href = '/booth'`.
    *   Gunakan validasi JavaScript sederhana, tampilkan pesan error dengan gaya *Neo-Brutalism modal*.

### Fase 1: Setup Mode (Step 1)
*   **Halaman Utama:** `booth.astro` (Area `<section id="step-1">`)
*   **Fitur:** Mengatur konfigurasi sesi foto sebelum dimulai.
*   **Logika & State yang Disimpan (`sessionConfig` object):**
    1.  **Pilih Layout:** Classic Strip (3 susun), 2x2 Grid (4 kotak), atau Polaroid (2 kotak). (Memengaruhi jumlah foto yang akan diambil).
    2.  **Pilih Timer:** 3 detik (Cepat) atau 5 detik (Santai).
    3.  **Pilih Filter Dasar (Opsional di awal):** Normal, Hitam Putih, Sepia.
    4.  **Tombol "Start Photobooth Session":** Saat diklik, sembunyikan `#step-1`, tampilkan `#step-2`, jalankan fungsi inisialisasi kamera.

### Fase 2: Mesin Auto-Shoot Kamera (Step 2)
*   **Halaman Utama:** `booth.astro` (Area `<section id="step-2">`)
*   **Fitur:** Kamera berjalan otomatis berdasarkan `sessionConfig`. **TIDAK ADA TOMBOL CAPTURE MANUAL.**
*   **Logika (The Engine):**
    1.  Minta izin kamera. Tampilkan video *live feed*.
    2.  Tampilkan hitung mundur besar di tengah layar (sesuai timer yang dipilih).
    3.  Gunakan `setInterval` atau `setTimeout` rekursif.
    4.  **Aksi saat timer = 0:**
        *   Mainkan suara `shutter.mp3`.
        *   Berikan efek *flash screen* (div putih dengan `opacity` transisi cepat 1 ke 0).
        *   Gambar frame video saat itu ke canvas sementara, konversi ke *DataURL*, lalu masukkan ke array sementara (misal: `capturedPhotos[]`).
    5.  Jika panjang `capturedPhotos` belum mencapai target (misal target 3 dari *Classic Strip*), ulangi hitung mundur.
    6.  Jika selesai, hentikan *stream* kamera (`track.stop()`), sembunyikan `#step-2`, tampilkan `#step-3`.

### Fase 3: Customize & Penggabungan (Step 3)
*   **Halaman Utama:** `booth.astro` (Area `<section id="step-3">`)
*   **Fitur:** Merakit *Photo Strip*, mengganti filter terakhir, dan menambahkan teks (Caption).
*   **Logika (The Renderer):**
    1.  **Kanvas Utama (Master Canvas):** Buat canvas tersembunyi beresolusi tinggi (misal 1080px lebar).
    2.  **Rendering Base:** Gambar latar belakang (warna solid) sesuai tata letak.
    3.  **Looping & Cropping:** Loop array `capturedPhotos`. Berdasarkan koordinat X/Y yang sudah didefinisikan untuk layout yang dipilih (misal Strip 3), gambar foto-foto tersebut menggunakan `ctx.drawImage()` yang memperhitungkan *object-fit: cover*.
    4.  **Terapkan Filter:** Jika user memilih filter baru dari tombol di bawah preview, bersihkan canvas utama, set `ctx.filter`, dan gambar ulang proses #3.
    5.  **Overlay Frame PNG (Opsional):** Jika layout memiliki bingkai bertekstur (dari folder `/public/frames`), gambar bingkai tersebut di atas segalanya (z-index teratas).
    6.  **Input Caption:** Sediakan text input "Add a caption (optional)". Jika diketik, jalankan fungsi render ulang yang menambahkan `ctx.fillText(text, x, y)` di area bawah *Photo Strip*.
    7.  Ubah canvas hasil akhir ini menjadi DataURL dan tampilkan di tag `<img>` sebagai *Live Preview*.

### Fase 4: Ekspor & Download (Step 4)
*   **Halaman Utama:** `booth.astro` (Area `<section id="step-4">`)
*   **Fitur:** Menyimpan hasil akhir ke perangkat pengguna.
*   **Logika:**
    1.  Tampilkan pesan "Mission Complete!" bergaya Arcade.
    2.  Sediakan dua tombol *Download* menggunakan atribut `<a> download`.
    3.  **Tombol 1 (Save Original):** Langsung mengunduh kanvas *Photo Strip* dari Step 3 (Cocok untuk dicetak).
    4.  **Tombol 2 (Save for Social):** 
        *   Buat `<canvas>` baru berukuran 1080x1350 (atau 1080x1920).
        *   Isi dengan warna/pattern latar belakang yang keren.
        *   Gambar gambar *Photo Strip* (dari Step 3) di tengah-tengah canvas baru ini.
        *   Unduh canvas baru ini (Cocok untuk Instagram Story/Feed).
    5.  Sediakan tombol "Replay Mission" untuk mereset seluruh *state* (kosongkan `capturedPhotos`), hapus preview, dan kembali ke Step 1.
