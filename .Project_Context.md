# 📸 MASTER BLUEPRINT: HYBRID NEO-BRUTALISM PHOTOBOOTH (CLIENT-SIDE SPA)

## .Project_Context
**Visi Produk:** Membangun aplikasi web Photobooth murni Client-Side (tanpa database/backend) yang menggabungkan dua identitas:
1.  **Visual Identity (Dari Ref 1 - Pixenze):** Gaya "Neo-Brutalism / Arcade Game". Ciri khas: Warna kontras cerah (Kuning, Merah, Biru Tua, Hijau Neon), *border* hitam tebal (3px-4px), *solid block shadow* yang tidak blur (`4px 4px 0px #000`), font *chunky/pixelated*, dan elemen UI menyerupai menu *video game*.
2.  **User Experience (Dari Ref 2 - Jelly's):** Alur "Automated Stepper/Wizard". Alih-alih memotret manual satu per satu, pengguna mengatur sesi di awal (Setup), lalu kamera memotret secara otomatis sesuai interval (Auto-Shoot), diikuti fase kustomisasi akhir (menambahkan teks/filter), lalu unduh.

**Batasan Mutlak (Hard Constraints):**
- **100% Local Processing:** Gunakan HTML5 `<canvas>` untuk menggabungkan gambar. Tidak boleh ada API *upload* ke server.
- **Framework:** Astro (sebagai *Static Site Generator* dan *Router* utama).
- **Styling:** Tailwind CSS (Custom config wajib untuk warna dan shadow Neo-Brutalism).
- **Interactivity:** Vanilla JavaScript di dalam tag `<script>` Astro. Tidak menggunakan React/Vue.

## .Features_Detail (Spesifikasi Fitur Ekstraksi Referensi)

### A. Fitur Global & Static Pages (Dari Ref 1)
- **Passcode Gate (`index.astro`):** Layar input PIN. Terdapat UI tombol "Start Game". Jika PIN salah, muncul Modal Error warna merah dengan efek *shake*.
- **Static Pages:** `about.astro`, `privacy.astro`, `contact.astro`. Semuanya menggunakan desain "Card" dengan border hitam tebal berlatar biru tua dotted (titik-titik).
- **Donate / QRIS Modal:** Modal pop-up (warna kuning cerah) untuk donasi, muncul tanpa merubah halaman.

### B. Fitur Main App SPA - "The Stepper" (Dari Ref 2 dibalut visual Ref 1)
- **State Management Object:** Variabel global JS `let sessionConfig = { poses: 3, timer: 3, layout: 'strip', filter: 'none', photos: [] }`.

- **Step 1: SETUP MISSION (Setup Phase)**
  - UI berupa kotak-kotak *selectable card* dengan efek *pressed* saat dipilih.
  - **Pilih Jumlah Pose:** 3 Poses atau 4 Poses.
  - **Pilih Timer:** 3 Detik (Cepat) atau 5 Detik (Santai).
  - **Pilih Layout:** Classic Strip (Vertikal) atau 2x2 Grid (Kotak).
  - Tombol aksi: "START CAMERA ENGINE 🚀".

- **Step 2: SHOOTING PHASE (Auto-Camera Engine)**
  - Menampilkan `<video autoplay>` dengan efek *CRT scanline* tipis (CSS overlay).
  - Terdapat UI "Inventory Slot" kosong di sebelah kanan (berjumlah 3 atau 4 sesuai pilihan Step 1).
  - Animasi hitung mundur besar di tengah layar dengan suara *beep*.
  - *Flash effect* (layar putih kilat 0.1 detik) + suara *shutter* saat jepret.
  - Foto yang dijepret langsung masuk mengisi "Inventory Slot" di UI.

- **Step 3: CUSTOMIZATION (Render Phase)**
  - Menampilkan *Live Preview* dari `<canvas>` yang sudah menggabungkan foto-foto dengan bingkai (Frame PNG).
  - **Live Filters:** Deretan tombol (Normal, B&W, Sepia, Retro/Warm). Jika diklik, canvas dirender ulang dengan `ctx.filter`.
  - **Caption Feature:** Input teks `maxlength="30"`. Jika diisi, tulis teks tersebut menggunakan `ctx.fillText` di bagian bawah *Photo Strip* dengan font gaya tulisan tangan atau mesin tik.

- **Step 4: MISSION COMPLETE (Export Phase)**
  - Menampilkan hasil akhir.
  - **Tombol "Save to Disk" (Print):** Unduh resolusi asli kanvas panjang (misal 600x1800 px).
  - **Tombol "Save for Social" (IG Story):** Unduh kanvas khusus rasio 9:16 (1080x1920 px) di mana *Photo Strip* diletakkan di tengah dengan background motif *star/dots* retro.
  - Tombol "Replay Mission" (Merestart array `photos` dan kembali ke Step 1).

## .rules (Core Workflows & SPA Logic)

Ini adalah alur logika JavaScript murni yang harus diterapkan di dalam `booth.astro` tanpa me- *reload* halaman.

1. **SPA Toggling Rule:**
   - Gunakan `<section id="step-1" class="block">`, `<section id="step-2" class="hidden">`, dst.
   - Buat fungsi JS `function goToStep(stepNumber)` yang memanipulasi class `hidden` Tailwind untuk perpindahan mulus.

2. **Auto-Shoot Logic Rule (`startShooting()`):**
   - Buat fungsi rekursif atau gunakan `async/await` dengan `setTimeout`.
   - Pseudo-code:
     ```javascript
     for(let i=0; i < sessionConfig.poses; i++) {
        await startCountdown(sessionConfig.timer); // 3..2..1
        playFlashAndSound();
        captureToCanvasArray(); // drawImage dari <video> ke temporary <canvas>
        updateInventoryUI(i);
     }
     stopCameraStream();
     goToStep(3);
     ```

3. **Canvas Rendering Rule (`renderFinalStrip()`):**
   - **Urutan Penggambaran (Layering/Z-Index Canvas):**
     1. Tentukan ukuran `MasterCanvas` (berdasarkan Layout).
     2. `ctx.fillStyle = "#FFF8E7"` (Gambar background warna krem dasar).
     3. *Loop* array `sessionConfig.photos`. Untuk setiap foto:
        - Terapkan `ctx.filter` sesuai pilihan user.
        - Gunakan metode *crop object-fit cover* dalam `ctx.drawImage(img, sx, sy, sw, sh, dx, dy, dw, dh)` agar foto (16:9) pas masuk ke dalam lubang bingkai (misal 4:3) tanpa menjadi "gepeng".
     4. Reset filter `ctx.filter = 'none'`.
     5. Gambar overlay bingkai PNG di atasnya `ctx.drawImage(frameImage, 0, 0)`.
     6. Gambar teks caption jika ada `ctx.fillText(...)`.

## .skill (Required Agent Capabilities & Error Prevention)

1. **Tailwind Neo-Brutalism Architecture:**
   - Agent harus membuat `tailwind.config.mjs` yang mendefinisikan *custom utility* kelas:
     ```javascript
     boxShadow: {
       'neo': '4px 4px 0px 0px #000000',
       'neo-sm': '2px 2px 0px 0px #000000',
     }
     ```
   - Agent harus memanipulasi transisi tombol (saat `:active` atau ditekan, *translate* elemen sejauh 4px ke bawah/kanan dan hilangkan bayangan untuk memberi efek tombol fisik).

2. **WebRTC API Security & Handling:**
   - Akses `navigator.mediaDevices.getUserMedia` hanya berfungsi di konteks yang aman (HTTPS atau localhost). Agent harus menangani error (`catch`) jika user menolak izin kamera, dan menampilkan UI Modal "Camera Access Denied".
   - Kamera **wajib** di- *mirror* (`transform: scaleX(-1)`) di CSS, dan secara matematis di- *mirror* juga saat dipindah ke Canvas (`ctx.translate(canvas.width, 0); ctx.scale(-1, 1);`) agar hasil unduhan sama dengan tampilan preview.

3. **Memory Management (Mobile Optimization):**
   - Array `sessionConfig.photos` harus menyimpan data sebagai elemen `<canvas>` sementara atau object `Image`, hindari menyimpan *Base64 String* ukuran penuh sebanyak 6 kali di RAM secara bersamaan karena browser Safari iOS sering memicu error *out-of-memory*.
   - Segera lepaskan `MediaStreamTrack` (`track.stop()`) begitu fase `Step 2` selesai, agar lampu indikator kamera laptop/HP mati saat user masuk ke fase *Customize*.

4. **Cors & Canvas Taint:**
   - Karena aplikasi bersifat lokal, pastikan pemanggilan gambar bingkai dari `/public/frames/` tidak menyebabkan *Tainted Canvas Error*. Jika perlu, load gambar dengan `img.crossOrigin = "Anonymous"`.
